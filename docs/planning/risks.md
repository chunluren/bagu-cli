## 风险评分

```
影响（Impact）
  1 = 微小（不影响进度）
  2 = 一般（拖延 1 周）
  3 = 中等（拖延 2-4 周）
  4 = 大（影响里程碑）
  5 = 灾难（项目失败）

概率（Probability）
  1 = < 10%
  2 = 10-30%
  3 = 30-50%
  4 = 50-80%
  5 = > 80%

风险分 = Impact × Probability
分级：
  1-4   低（监控）
  5-12  中（缓解）
  13-25 高（必须处理）
```

## 当前风险（2026-04-29）

### R001 — 时间不足（求职冲突） 🔴

| 字段 | 值 |
|----|----|
| **影响** | 4（项目无法完成） |
| **概率** | 4（春招 + 求职面试 + 写代码冲突）|
| **分数** | 16 |
| **状态** | 高，必须缓解 |
| **责任人** | 作者 |

**说明：**
作者春招期间需要刷题、面试、修简历，业余时间有限。

**缓解措施：**
- 优先 MVP，砍 P2/P3 功能
- 每天固定 1-2 小时投入
- 面试季可暂停 1-2 周
- 关键功能先做，质量后补

**触发指标：**
- 连续 1 周无 commit
- 里程碑延后超过 1 周
