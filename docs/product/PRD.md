## 1. 产品概述

### 1.1 一句话定义
**bagu-cli** 是一个本地命令行工具，把零散的 markdown 八股文档转化为可交互的复习与面试模拟系统。

### 1.2 解决什么问题

| 痛点                            | 当前方案的不足                  | bagu-cli 怎么解决             |
|--------------------------------|-------------------------------|------------------------------|
| 八股文档过长，反复看疲劳         | typora / vscode 静态阅读       | 结构化抽取 + 主动推送复习       |
| 不知道哪个知识点没掌握           | 无客观反馈                     | SM-2 算法跟踪掌握度            |
| 需要模拟问答练习                | Anki 太重 + 不会针对性提问      | AI 模拟面试 + 自适应难度       |
| 笔试前临时抱佛脚找不到点         | grep 八股文档不直观            | 内置全文搜索 + tag             |
| 复习无规划                      | 完全凭感觉                     | 每日推送 + 进度可视化           |

### 1.3 不做什么（明确边界）

- ❌ **不做笔记编辑** — 用户用 typora 等编辑 md，本工具只读
- ❌ **不做云同步** — 数据本地存储，不上传
- ❌ **不做 GUI** — 只做 CLI/TUI
- ❌ **不做内容生产** — 八股内容由用户/作者维护
- ❌ **不做账号体系** — 单机单用户
- ❌ **不做付费/订阅** — 完全开源免费

## 2. 用户与场景

### 2.1 目标用户

详见 [用户画像](./personas.md)。

**核心用户：**
- 准备校招 / 社招面试的程序员（C++ / Java / Go 后端为主）
- 已有自己的 markdown 八股文档库

### 2.2 用户旅程（典型一周）

```
周一晚上（30 分钟）  → bagu review        日常复习昨日卡片
周二午休（10 分钟）  → bagu search "MVCC" 临时查问题
周三晚上（45 分钟）  → bagu interview      AI 模拟面试一次
周四白天             → 上班/上课
周五晚上（30 分钟）  → bagu review        + bagu stats 看周报
周六                 → 学新章节，导入新文档（bagu import）
周日                 → 复盘 + 调整
```

## 3. 功能需求

### 3.1 功能清单（按优先级）

#### P0 - MVP 必须
| 功能          | 描述                                     | 验收标准                           |
|--------------|----------------------------------------|----------------------------------|
| `init`       | 初始化数据目录、写默认配置                  | 能创建 ~/.bagu/ 与 db schema       |
| `import`     | 导入 markdown 文档到本地数据库              | 支持目录递归，处理重复导入            |
| `list`       | 列出主题与章节                              | 树形显示，含题数                    |
| `search`     | 全文搜索题目                                | 模糊匹配 + 关键词高亮               |
| `review`     | 进入复习模式（SM-2 算法）                    | TUI 流畅，按键评分，进度持久化        |

#### P1 - 重要
| 功能            | 描述                          | 验收标准                |
|----------------|------------------------------|------------------------|
| `interview`    | AI 模拟面试                   | 接 OpenAI/Claude/Ollama |
| `stats`        | 学习进度统计                  | 总览 + 各主题            |
| `stats --heatmap` | GitHub 风格热力图            | 90 天复习活动            |
| `config`       | 配置读写                       | get/set 子命令          |

#### P2 - 增强
| 功能            | 描述                          |
|----------------|------------------------------|
| `export anki`  | 导出 Anki 牌组                 |
| `tag`          | 给卡片加 tag                   |
| `bookmark`     | 收藏卡片                       |
| `note`         | 在卡片上记自己的笔记             |

#### P3 - 探索
| 功能            | 描述                          |
|----------------|------------------------------|
| Web UI         | 本地 web 界面（可选）          |
| 多端同步        | 加密同步到 GitHub Gist         |
| 团队共享        | 多人分享卡片库                 |

### 3.2 详细功能规范

每个命令的详细 spec 见 [CLI 命令规范](../architecture/cli-spec.md)。

## 4. 非功能需求（NFR）

### 4.1 性能

| 指标              | 目标           | 极限           |
|-----------------|---------------|---------------|
| CLI 启动时间      | < 50 ms       | < 100 ms      |
| 搜索响应          | < 100 ms      | < 500 ms      |
| 导入速度          | 1000 cards/s  | > 500 cards/s |
| 单次复习题加载     | < 10 ms       | < 50 ms       |
| LLM 流式响应延迟   | < 1 s 首字     | < 3 s         |
| 内存占用（运行时）  | < 50 MB       | < 100 MB      |

详见 [性能预算](../quality/performance-budget.md)。

### 4.2 可用性

- **跨平台**：Linux（主要） / macOS（次要） / Windows（best effort）
- **零配置启动**：`bagu init` 后立即可用
- **离线优先**：除 LLM 外不联网

### 4.3 可靠性

- 数据库操作必须事务性（ACID）
- 复习记录不能因为崩溃丢失
- 单元测试覆盖率 ≥ 80%
- 关键路径覆盖率 ≥ 95%

### 4.4 可维护性

- 严格遵循 [编码规范](../development/coding-standards.md)
- 模块解耦，依赖单向（CLI → core → service → db）
- 所有公开 API 有 doxygen 注释

### 4.5 安全

- 不收集任何遥测
- 配置文件中不存明文 API key（用环境变量）
- 数据库本地存储，不上传
- 详见 [安全清单](../quality/security-checklist.md)

### 4.6 国际化

- v1.0 中英文混合（错误信息英文，提示文字简体中文）
- v1.1 考虑加 i18n 框架

## 5. 成功度量（KPI）

### 5.1 自用指标（最重要）

- 我自己每周用 ≥ 3 次
- 自己 4 周内通过 bagu 复习 ≥ 200 张卡片
- 春招期间至少 1 次面试用到 bagu 学的内容

### 5.2 开源指标

- 6 个月内 GitHub ≥ 100 star
- 3 个月内有外部用户 issue / PR

### 5.3 质量指标

- 测试覆盖率 ≥ 80%
- 0 个 P0 级 bug 进入主分支
- CI 失败率 < 5%

## 7. 假设与依赖

### 7.1 假设

- 用户已有 markdown 八股文档（或愿意写）
- 用户熟悉命令行操作
- 用户有 Linux/Mac 环境

### 7.2 依赖

- C++20 编译器（GCC 10+ / Clang 13+）
- CMake 3.20+
- 网络访问（CMake FetchContent 拉依赖）
- LLM API key（仅 interview 功能需要）
