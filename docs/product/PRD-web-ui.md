## 1. 概述

### 1.1 一句话定义

为 bagu-cli 增加 **本地 HTTP 服务模式** + **轻量 Web 前端**，让用户在浏览器（含手机浏览器）中使用 bagu-cli 的全部能力，数据仍 100% 本地。

### 1.2 解决什么问题

| 痛点 | 现状 | Web UI 怎么解决 |
|----|----|----|
| 终端阅读长答案累 | TUI 翻页 / `bagu show <id>` | 浏览器富文本渲染（含代码高亮） |
| 在手机上想复习 | 只能 SSH 到电脑 | 同 WiFi 浏览器访问，体验自然 |
| 想边写代码边复习 | 切换终端 | 浏览器开个 tab 即可 |
| 截图分享不方便 | TUI 含 ANSI 难截 | 浏览器一键截屏 |
| 与 ChatGPT / Claude 对照看 | 多终端切换 | 浏览器并排开 |

### 1.3 不做什么

- ❌ **不做云同步** — 数据仍 SQLite 本地存储
- ❌ **不做账号系统** — 单机单用户
- ❌ **不做服务器部署** — 只跑本地 / 局域网
- ❌ **不做内容编辑** — Web UI 只读 + 复习；编辑卡片仍走 markdown 文件 + `bagu import`
- ❌ **不替代 CLI** — CLI 仍为主力，Web 仅为补充入口

## 3. 功能需求

### 3.1 P0（MVP，必做）

| 功能 | 详情 |
|----|----|
| `bagu serve` CLI 命令 | 启动 HTTP server，可指定端口、绑定地址 |
| REST API | 涵盖现有 CLI 的所有读路径 |
| 主题列表页 | 等同 `bagu list`（表格 / 卡片视图） |
| 章节树页 | 等同 `bagu list <topic>` |
| 卡片详情页 | 等同 `bagu show <id>`，含 markdown 渲染 |
| 全文搜索 | 等同 `bagu search`，关键词高亮 |
| 复习页 | 等同 `bagu review`，但用浏览器键盘 + 触屏评分 |

### 3.2 P1（重要）

| 功能 | 详情 |
|----|----|
| 学习统计页 | 等同 `bagu stats`，含可点击的热力图 |
| AI 模拟面试页 | 等同 `bagu interview`，含流式输出（SSE） |
| 全局配置页 | 等同 `bagu config`，可视化编辑 |
| 暗黑/明亮主题 | 跟随系统 |

### 3.3 P2（可选）

| 功能 | 详情 |
|----|----|
| 收藏夹 | 给卡片加 ⭐ |
| 笔记 | 在卡片上记自己的批注（写入本地 db） |
| 导出 | 一键导出 Anki 牌组 / PDF |
| 多窗口对照 | 一边题目一边答案对比 |

### 3.4 P3（不做）

- 多用户认证、协作、评论
- 推送通知
- 移动端原生 App

## 4. 非功能需求（NFR）

### 4.1 性能

| 指标 | 目标 | SLO |
|----|----|----|
| `bagu serve` 启动 | < 100ms | < 300ms |
| 首页加载（冷） | < 500ms | < 1s |
| API 响应（含 db） | < 50ms | < 200ms |
| 搜索响应 | < 100ms | < 500ms |
| 流式 AI 首字节 | < 1s | < 3s（取决于 LLM） |

### 4.2 资源消耗

- 内存：常驻 < 100 MB
- CPU 空闲：< 1%
- 端口：默认 8780（避开常用端口）

### 4.3 安全

- **默认绑定 127.0.0.1**（仅本机访问）
- 可选 `--bind 0.0.0.0` 允许局域网（要警告）
- 可选 `--token <SECRET>` 启用 Bearer 鉴权
- 不写入除 SQLite / 日志外的任何文件
- CORS 只允许同源 + localhost
- 不依赖任何 CDN（前端资源全部内嵌或同域分发）

### 4.4 可靠性

- 后端崩溃不丢失数据（SQLite WAL）
- API 错误返回标准 JSON：`{ "error": { "code": E1234, "message": "..." } }`
- 前端断网时降级显示

### 4.5 可访问性

- 键盘可完全操作（j/k 翻页、空格揭示、1-5 评分）
- 颜色对比度满足 WCAG AA
- 移动端友好（视口适配 / 触屏评分按钮足够大）

## 5. 成功指标

### 5.1 自用指标

- 周内自己用 ≥ 2 次（手机或浏览器）
- 单次复习时长 ≥ 10 分钟
- 简历有 Web UI 截图

### 5.2 工程指标

- API 响应延迟 P99 < 200ms
- 前端 bundle < 300 KB（gzip）
- 单测覆盖率（后端 API）≥ 80%

## 7. 假设与依赖

### 7.1 假设

- 用户能在浏览器装 PWA / 把页面加到主屏（手机用）
- 用户和 bagu serve 在同一网络（家庭 WiFi / Tailscale 等）

### 7.2 依赖

- **后端**：cpp-httplib（[ADR-0004](../architecture/adr/0004-add-http-server-mode.md) 决定）
- **前端**：Vite + React + TypeScript（[ADR-0006](../architecture/adr/0006-frontend-stack-vite-react.md) 决定）
- **markdown 渲染**：marked.js + highlight.js
- **图表（热力图）**：自实现 SVG（不引第三方）

## 8. 风险

详见 [risks.md](../planning/risks.md) 的新增条目（R011-R014）。

| 风险 | 概率 | 影响 |
|----|----|----|
| HTTP server 与 CLI 同时操作 db 冲突 | 低 | 中（WAL 已支持并发读 + 单写） |
| 前端 bundle 过大影响首屏 | 中 | 中（Vite + 按需加载） |
| 移动端体验不达标 | 中 | 中（响应式 + 触屏适配） |
| 未来想做云同步又要重构 | 低 | 高（先在 API 设计上预留） |
