# 用户画像

> **状态**：Approved
> **更新**：2026-04-29

## Persona 1：求职应届生（核心用户）

**名字**：李同学，男，25 岁，硕士在读

**背景**：
- 计算机硕士在读，2026 届毕业
- 准备 C++ 后端开发岗位
- 平时学习用 typora 写 markdown 笔记
- 有自己整理的 8 份八股文档（mysql / redis / 网络等）

**目标**：
- 4 个月内通过校招 / 春招拿到 offer
- 系统化复习已写的八股
- 能模拟面试场景练习答题

**痛点**：
- 八股文档太长，被动阅读效率低
- 不知道哪些章节没掌握
- 临考前难找重点
- 没有便捷的提问 - 答题循环

**使用场景**：
- 每晚 30-45 分钟复习
- 周末整理新内容
- 面试前一天临时抱佛脚
- 答错的题反复刷

**对工具的期待**：
- 启动快、不卡
- 不需要联网就能用
- 不要太花哨，能专心
- 数据在自己手里

**反对什么**：
- 强制注册账号
- 要付费
- 闪屏 / 弹广告
- 复杂的导入流程

## Persona 2：在职转岗者

**名字**：陈工，男，28 岁，工作 3 年

**背景**：
- 当前 Java 后端，想转 C++
- 业余时间学习
- 有一定基础但 C++ 八股不熟

**目标**：
- 3-6 个月内完成八股扫盲
- 准备社招面试

**痛点**：
- 时间碎片化（早晚通勤）
- 不知道学到哪算够用
- 没有人帮自己评估水平

**期待**：
- 能在手机上看（暂不支持，但 SSH 终端连家里电脑可用）
- 能快速回顾昨天看过的内容
- AI 能针对性提问

## Persona 3：开源用户（潜在）

**名字**：开源社区用户

**背景**：
- 通过 GitHub 发现项目
- 程序员，自己也写八股

**目标**：
- 评估是否值得用
- 自己 fork 来加功能

**对项目的期待**：
- README 清晰
- 编译简单
- 文档完整
- 代码质量过得去

## Anti-Persona（明确不服务）

❌ **小白用户** — 不会编译 C++ 的不是目标
❌ **教培行业** — 不做内容生产平台
❌ **企业客户** — 不做 SaaS / 团队版
