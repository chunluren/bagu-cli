# ADR-0001: 选用 C++20

- **状态**: Accepted
- **日期**: 2026-04-29
- **作者**: Li Yan

## 背景

bagu-cli 是新项目，需要确定 C++ 标准版本。

约束：
- 求职项目，希望体现现代 C++ 能力
- 主流 Linux 发行版（Ubuntu 22.04+）的 GCC 默认 11，支持 C++20
- macOS Clang 14+ 支持
- Windows MSVC 2022 支持

## 决策

**使用 C++20。**

启用的关键特性：
- `std::format`（部分实现，需 GCC 13+）
- Concepts
- Ranges
- `std::span`
- `consteval` / `constinit`
- 改进的 `<chrono>`
- 协程（不强求，按需）

## 备选方案

| 方案    | 优点                          | 缺点                            | 不选理由 |
|-------|------------------------------|-------------------------------|------|
| C++14 | 兼容性最好                       | 无现代特性                       | 太老  |
| C++17 | structured bindings、optional 等成熟 | 没有 ranges/concepts/format | 已有更好选择 |
| C++23 | 最新特性                         | 编译器支持不全                    | 风险高 |

## 后果

### 正面

- 简历加分（"使用 C++20 现代特性"）
- 代码更简洁（ranges、concepts）
- 标准库更强大

### 负面

- 老编译器（GCC 9 及以下）不支持 → 用户需要升级
- 部分特性如 `std::format` 在 GCC 13 才完整 → 必要时手写 `fmt`

### 中性

- 需要在 README 标明编译器最低版本
- CI 至少跑 GCC 11 / Clang 14

## 实施

CMakeLists.txt：
```cmake
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)
```
