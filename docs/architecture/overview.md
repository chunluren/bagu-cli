## 1. 设计目标

| 维度        | 目标                                    |
|-----------|---------------------------------------|
| 性能       | 启动 < 50ms、搜索 < 100ms                |
| 可维护     | 模块解耦、单向依赖、清晰职责                |
| 可测试     | 80%+ 覆盖率，纯函数为主                   |
| 跨平台     | Linux / macOS（Windows best effort）   |
| 零配置     | 一行命令初始化                            |
| 离线优先   | 除 LLM API 外不联网                      |

## 2. 总体架构（4+1 视图：逻辑视图）

```
┌──────────────────────────────────────────────────────────┐
│                    展现层（Presentation）                  │
├──────────────────────────────────────────────────────────┤
│  ┌────────────┐  ┌─────────────┐  ┌──────────────────┐  │
│  │  CLI 入口   │  │  TUI 界面    │  │  错误码 / 输出格式 │  │
│  │  (CLI11)   │  │  (FTXUI)    │  │                   │  │
│  └────────────┘  └─────────────┘  └──────────────────┘  │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│                    应用层（Application）                   │
├──────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │ ImportSvc │ │ ReviewSvc │ │ SearchSvc │ │InterviewSvc│ │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘ │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│                    领域层（Domain）                        │
├──────────────────────────────────────────────────────────┤
│  ┌────────────┐ ┌────────────┐ ┌────────────────────┐   │
│  │ SM-2 算法   │ │ 评分系统    │ │ 卡片选择策略         │   │
│  └────────────┘ └────────────┘ └────────────────────┘   │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│                    基础设施层（Infrastructure）             │
├──────────────────────────────────────────────────────────┤
│  ┌────────┐ ┌─────────┐ ┌─────────┐ ┌──────┐ ┌────────┐ │
│  │ DB(SQLite) │Markdown │FTS Search│ HTTP │Config(TOML)│ │
│  │            │ Parser  │ Engine   │Client│           │ │
│  │            │(cmark)  │(SQLite   │(curl)│           │ │
│  │            │         │ FTS5)    │      │           │ │
│  └────────┘ └─────────┘ └─────────┘ └──────┘ └────────┘ │
└──────────────────────────────────────────────────────────┘
                          ↓
┌──────────────────────────────────────────────────────────┐
│                    跨切面（Cross-cutting）                  │
├──────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐ │
│  │  Logging │  │  Errors  │  │  Metrics  │  │   Util   │ │
│  │ (spdlog) │  │  (Result)│  │ (counters)│  │ (string) │ │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘ │
└──────────────────────────────────────────────────────────┘
```

### 2.1 分层原则

- **依赖单向**：上层依赖下层，下层不知道上层
- **接口先行**：层间通过抽象接口交互
- **跨层禁止**：禁止 CLI 直接访问 DB
- **跨切面无状态**：logging/errors 等可被任意层使用

### 2.2 数据流（一次 review 命令）

```
用户：bagu review
  ↓
CLI 解析（CLI11）
  ↓
ReviewService.start(profile)
  ↓
CardRepository.fetchDueCards()  → SQLite SELECT
  ↓
SM2Algorithm.pickNext(cards)
  ↓
TUI.show(card)  → FTXUI 渲染
  ↓
用户按键 1-5
  ↓
SM2Algorithm.update(card, score)
  ↓
ReviewRepository.save(record)  → SQLite INSERT/UPDATE
  ↓
循环 / 退出
```

## 3. 模块边界

| 模块        | 路径               | 职责                              | 依赖                     |
|-----------|------------------|---------------------------------|--------------------------|
| cli       | src/cli/         | 命令解析、参数校验、子命令分发           | core, util               |
| tui       | src/tui/         | 终端 UI 渲染与交互                   | core, util               |
| core      | src/core/        | 领域逻辑（SM-2、评分、卡片选择）          | db                       |
| service   | src/service/     | 应用层服务（编排领域逻辑）              | core, db, parser, llm    |
| db        | src/db/          | SQLite 封装、DAO                   | (无)                     |
| parser    | src/parser/      | Markdown 解析                      | (无)                     |
| search    | src/search/      | 全文搜索（基于 FTS5）                | db                       |
| llm       | src/llm/         | LLM API 客户端（OpenAI/Claude/Ollama）| util(http)               |
| util      | src/util/        | 通用工具（字符串、HTTP、文件、time）     | (无)                     |

### 3.1 依赖图（无环）

```
        cli ───────┐

         ▼         ▼
        service   tui
       /  │  \    │
      ▼   ▼   ▼   │
   core  parser  llm
      \    │    /
       ▼   ▼   ▼
        db / util
```

## 4. 关键技术决策

详细决策见 [ADR](./adr/README.md)，核心选择：

| 决策             | 选择              | 理由                                |
|----------------|------------------|----------------------------------|
| 语言             | C++20            | 现代特性、求职岗位主流                  |
| 构建             | CMake + FetchContent | 零依赖、可移植                      |
| 存储             | SQLite + FTS5    | 轻量、SQL 完整、内置全文搜索            |
| CLI 解析         | CLI11            | 现代 C++ 风格                       |
| TUI             | FTXUI            | 声明式、跨平台                       |
| HTTP            | libcurl          | 工业标准                            |
| Markdown 解析    | cmark            | CommonMark 标准、稳定                |
| 配置             | toml++           | TOML 比 JSON 适合人写                |
| 测试             | GoogleTest       | 业界标准                            |
| 日志             | spdlog           | 现代、性能好                         |

## 6. 质量属性策略

### 6.1 性能

| 属性     | 策略                                     |
|--------|----------------------------------------|
| 启动快   | 延迟初始化（不用的模块不加载）              |
| 搜索快   | SQLite FTS5 倒排索引                     |
| 响应快   | TUI 用增量重绘（FTXUI 默认支持）            |

### 6.2 可靠性

- 所有 DB 操作用事务
- 关键操作有 unit test 覆盖
- 错误用 `Result<T>` 显式传播，不抛异常
- 崩溃后下次启动自检 db schema

### 6.3 可观测性

- spdlog 日志，按级别过滤
- 关键操作（导入、复习提交）有耗时日志
- `bagu --verbose` 开启详细日志

### 6.4 安全

- API key 仅从环境变量读取
- 数据库无明文敏感信息
- 输入校验（命令注入、SQL 注入）

## 7. 演进策略

### 7.1 版本兼容

- 数据库 schema 用 migration 脚本（src/db/migrations/）
- CLI 命令向后兼容（不删除已有命令）
- 配置文件加字段时给默认值

### 7.2 扩展点

- **新 LLM 提供商**：实现 `LLMClient` 接口
- **新文档格式**：实现 `Parser` 接口
- **新复习算法**：实现 `Scheduler` 接口

## 8. 不做的设计选择（Trade-offs）

| 不做                | 理由                                |
|------------------|----------------------------------|
| 微服务架构          | 单机工具不需要                       |
| ORM              | SQL 直接写更简单可控                  |
| 异步 / 协程        | CLI 工具同步代码够用                  |
| 插件机制           | 复杂度爆炸，先不做                    |
| 多用户            | 个人工具                            |
| 加密存储           | 数据本地，威胁模型无此需求              |
